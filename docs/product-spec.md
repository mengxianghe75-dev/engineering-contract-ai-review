# 产品规格说明

## 一、产品定位

本系统是一个面向工程合同场景的"智能审查与管理平台"原型产品。

它当前不是单纯的合同解析工具，而是已经具备：

- 多角色登录
- 文档管理
- 规则管理
- 上传版本管理
- 操作日志留痕

可以用于：

- 直播演示
- 客户原型汇报
- 内部联调
- 后续商业化升级基础版本

## 二、当前已实现能力

### 1. 登录与权限

- JWT 登录
- 角色控制：`admin / reviewer / viewer`
- 菜单、页面、接口三层权限控制

### 2. 合同管理

- PDF 上传
- 文本型 PDF 直接提取
- 扫描型 PDF OCR 识别
- 后台 OCR 队列
- 合同列表筛选
- 合同详情查看
- 归属、分类、标签、状态、归档

### 3. 审查能力

- 基础字段提取
- 风险识别
- 规则引擎优先
- AI 逻辑补充
- 审查摘要生成
- 支持 `mock` 与 OpenAI 兼容模型两种摘要模式
- 支持 OpenAI 兼容模型补充结构化风险项
- 审查报告导出

### 4. 规则管理

- 规则列表
- 新增规则
- 编辑规则
- 启用/停用
- 软删除
- 支持 `keyword / regex / contains_any / contains_all`

### 5. 上传版本

- 同一业务合同支持多次上传 PDF，形成 `V1 / V2 / V3` 上传版本
- 可查看上传版本列表
- 可对比两个上传版本的字段变化和风险变化
- 可导出指定上传版本报告
- 文件内审查快照会保存当次摘要调用状态和风险补充调用状态

### 6. 操作日志

已记录关键动作：

- 上传合同
- 编辑合同
- 触发审查
- 导出报告
- 新增规则
- 编辑规则
- 启用/停用规则
- 新增用户
- 编辑用户
- 启用/停用用户

## 四、文档管理规则

每份合同文档当前支持以下属性：

- `owner_id`
- `category`
- `tags`
- `status`
- `archived_at`
- `updated_by`

状态枚举：

- `uploaded`
- `parsed`
- `reviewed`
- `archived`

边界约定：

- 当前阶段不提供删除入口
- 后续建议支持软删除，不建议硬删除
- 归档后仍可查看和导出
- 归档后默认不直接进入重审

标签策略：

- 当前阶段使用自由输入
- 存储上采用字符串数组
- 后续可升级为固定词典或"词典 + 自由输入"

## 五、审查规则策略

规则执行顺序：

1. 读取启用规则
2. 按优先级排序
3. 执行规则匹配
4. 产出规则风险项
5. 再执行 AI 风险识别
6. 如已配置真实模型，再补充 LLM 风险识别
7. 合并结果，规则优先

命中信息：

- 保存 `rule_id`
- 保存 `matched_text`
- 保存 `match_detail`

历史结果：

- 规则禁用只影响未来
- 历史审查结果保留

## 六、上传版本策略

当前版本策略为两层：

- `contract_files` 表示实际上传的文件版本，同一业务合同可有多个上传版本
- `review_versions` 保存单个文件内部的审查历史快照
- `contract_review_results` 保存每个文件版本的最新审查结果

这样既方便对比两次上传文件差异，也保留了单个文件内部的审计留痕。

当前版本差异对比重点覆盖：

- 基础字段变化
- 新增风险
- 移除风险
- 风险等级变化

## 七、适合直播演示的价值点

建议重点强调：

- 不是单点工具，而是有后台管理能力的系统
- 大模型配置可在前端后台直接管理，不需要改代码
- 规则可以后台配置，不需要改代码
- 审查过程可留痕、可回看、可导出
- 没有接入复杂大模型时也能完整跑通
- 已支持平滑接入 OpenAI 兼容模型做摘要生成
- 已支持平滑接入 OpenAI 兼容模型做风险补充识别
- 后续可以继续扩展到更强的规则和 LLM 能力
